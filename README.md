# RAG Legal Assistant

## Введение

RAG Legal Assistant — это специализированный инструмент, разработанный для обработки и анализа российских юридических документов с использованием технологии Retrieval-Augmented Generation (RAG). Система позволяет пользователям:

1. Извлекать текст из юридических PDF-документов (таких как судебные решения и постановления)
2. Анализировать и структурировать извлеченную информацию на значимые компоненты (фабула дела, решение, ссылки на статьи законов)
3. Создавать фрагменты документов для эффективной обработки
4. Сохранять эти фрагменты в векторной базе данных (ChromaDB) с эмбеддингами для семантического поиска
5. Предоставлять интерфейс вопросов-ответов, где пользователи могут задавать вопросы по юридическим документам на естественном языке

Система специально ориентирована на российские юридические документы, что подтверждается наличием образца документа (постановление Арбитражного суда Московского округа) и функциями обработки на русском языке.

## Возможности

- **Извлечение данных из PDF**: Автоматическое извлечение текста из юридических PDF-документов
- **Интеллектуальная структуризация**: Разделение документов на логические секции (фабула, решение, нормативные акты)
- **Семантический поиск**: Поиск релевантной информации по смыслу, а не только по ключевым словам
- **Контекстно-зависимые ответы**: Генерация ответов с учетом контекста документа и истории диалога
- **Перефразирование вопросов**: Улучшение запросов пользователя для более точного поиска информации
- **Обработка русскоязычных документов**: Специализированные функции для работы с российскими юридическими текстами
- **Сохранение результатов**: Экспорт обработанных данных в структурированном JSON-формате

### Компоненты системы

1. **Слой загрузки данных**:
   - Загрузка и парсинг PDF с использованием `PyPDFLoader`
   - Пользовательское извлечение текста с функцией `extract_sections()`
   - Сохранение результатов парсинга в формате JSON

2. **Слой обработки текста**:
   - Фрагментация документов с помощью `RecursiveCharacterTextSplitter`
   - Преобразование в объекты `Document` LangChain с метаданными

3. **Слой векторной базы данных**:
   - Генерация эмбеддингов с использованием OpenAI
   - Хранение в ChromaDB для эффективного семантического поиска

4. **Слой извлечения данных**:
   - Извлечение контекста на основе запросов пользователя
   - Управление диалогом с учетом истории

5. **Слой генерации**:
   - Ответы на вопросы с использованием моделей OpenAI
   - Форматирование структурированных ответов

### Ключевые файлы и их назначение

- **RAG.ipynb**: Основной ноутбук, содержащий всю логику обработки документов и генерации ответов
- **parsed_output.json**: Хранит структурированные данные, извлеченные из PDF-документов
- **recursive_chunks.json**: Содержит фрагменты документов для обработки RAG
- **chroma_db/recursive_chunks/**: Директория с векторными представлениями фрагментов текста

## Установка и запуск

### Предварительные требования

- Python 3.12.6 (указано в `.python-version`)
- Ключ API OpenAI (хранится в файле `.env`)

### Шаги установки

1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/SergeyKarpenko1/RAG_Legal_assistant
   cd RAG_Legal_assistant
   ```

2. Создать виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # На Windows: venv\Scripts\activate
   ```

3. Установить зависимости (проект использует менеджер пакетов `uv`):
   ```bash
   pip install uv
   uv pip install -e .
   ```

4. Создать файл `.env` в корне проекта с вашим ключом API OpenAI:
   ```
   OPENAI_API_KEY="ваш-ключ-api-здесь"
   ```

5. Убедиться, что структура директории данных настроена правильно:
   ```bash
   mkdir -p Data/chroma_db/recursive_chunks
   ```

### Запуск

1. Запустить Jupyter Notebook:
   ```bash
   jupyter notebook
   ```

2. Открыть `Notebooks/RAG.ipynb`

3. Выполнить ячейки ноутбука последовательно для обработки документов и запуска интерфейса вопросов-ответов

## Как это работает

### Процесс обработки документов

1. **Загрузка PDF**:
   ```python
   # Загрузка PDF-документов из указанных директорий
   dirs = ["/путь/к/вашим/pdf"]
   out_json = "/путь/к/выходному.json"
   parse_pdf_and_save_json(dirs, out_json)
   ```

2. **Извлечение секций**:
   Система анализирует документ и извлекает ключевые секции, такие как фабула дела, решение суда и ссылки на законодательные акты.

3. **Создание фрагментов**:
   ```python
   # Создание фрагментов из обработанного документа
   text_splitter = RecursiveCharacterTextSplitter(chunk_size=400, chunk_overlap=50)
   chunks = text_splitter.split_text(document.page_content)
   ```

4. **Создание векторной базы данных**:
   ```python
   # Инициализация и заполнение ChromaDB
   embeddings = OpenAIEmbeddings(model="text-embedding-3-large")
   vectordb = Chroma(
       collection_name="recursive_chunks",
       embedding_function=embeddings,
       persist_directory="/путь/к/сохранению"
   )
   vectordb.add_documents(documents)
   ```

### Процесс ответа на вопросы

1. **Настройка ретривера**:
   ```python
   # Настройка ретривера
   retriever = vectordb.as_retriever(
       search_type="similarity",
       search_kwargs={"k": 5}
   )
   ```

2. **Инициализация чат-интерфейса**:
   ```python
   # Инициализация чат-интерфейса
   continual_chat()
   ```

3. **Пример взаимодействия**:
   ```
   ВЫ: Кто истец по этому делу?

   🔹 Оригинальный вопрос:
   Кто истец по этому делу?

   🔸 Перефразированный вопрос:
   Кто является истцом в данном деле?

   ✅ Ответ AI:
   Истцом в данном деле является ИП Фоменко Алексей Сергеевич.

   Актуальное решение суда:
   Кассационная жалоба ИП Фоменко была подана в связи с возвращением апелляционной жалобы из-за отклонения ходатайства о предоставлении отсрочки уплаты государственной пошлины.

   Основание для решения:
   - Арбитражный суд
   - Дело № А41-9245/2025
   - 20 мая 2025 года, Постановление
   - Нормы процессуального права (отсутствуют точные статьи)
   ```

4. **Внутренний процесс**:
   - Система перефразирует вопрос для оптимизации поиска
   - Извлекает релевантные фрагменты из векторной базы данных
   - Генерирует ответ на основе извлеченного контекста
   - Структурирует ответ с указанием источника информации

## Дополнительная информация

### Особенности обработки юридических документов

Система специально адаптирована для работы с российскими юридическими документами, включая:

- Распознавание структуры арбитражных решений
- Идентификацию фабулы дела, решения и нормативных актов
- Обработку специфической юридической терминологии
- Семантический поиск по юридическим концепциям

### Рекомендации по использованию

- Для достижения наилучших результатов рекомендуется использовать PDF-документы с качественным текстовым слоем
- Оптимальный размер фрагментов (chunk_size) может варьироваться в зависимости от структуры документов
- Для сложных документов может потребоваться увеличение количества извлекаемых фрагментов (параметр k)

### Ограничения

- Система основана на модели OpenAI, поэтому требует соединения с интернетом
- Точность ответов зависит от качества исходных документов
- Для обработки больших объемов документов может потребоваться оптимизация параметров
